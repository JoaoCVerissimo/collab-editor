The Magic

How CRDTs Prevent Conflicts

The secret ingredient that makes simultaneous editing work perfectly — explained simply.

🔬 What Is a CRDT?

A Conflict-free Replicated Data Type is a special kind of data structure. Multiple copies can be edited independently and, when brought back together, they always arrive at the same result — automatically.

🎯 Unique IDs for Everything

Every single keystroke gets a unique ID made from who typed it and when. This means even if two people type in the same spot at the same instant, the system can always tell the edits apart.

🔁 Deterministic Ordering

When two characters land at the same position, a simple, consistent rule decides the order (lower user ID goes first). Every computer applies the same rule, so everyone always sees the same result.

🔄 Order Doesn't Matter

The math behind CRDTs ensures that it doesn't matter what order the edits arrive. Apply them forwards, backwards, or jumbled — the final document is always identical everywhere.

✈️ Offline? No Problem.

When you go offline, your edits stack up locally. When you reconnect, only the missing edits are exchanged (using a small "state vector" summary). The CRDT merges everything perfectly.

✅ The Result

No "merge conflicts." No "someone else's changes overwrite yours." No "please resolve." Every edit from every user is preserved and woven together automatically and correctly.

API Reference

REST API Endpoints

The backend exposes a clean REST API for all non-realtime operations.

Authentication

MethodEndpointDescription
POST /api/auth/register Create a new account (email, display name, password)
POST /api/auth/login Sign in and receive a JWT token
GET /api/auth/me Get the currently signed-in user's profile

Documents

MethodEndpointDescription
GET /api/documents List all documents you own or collaborate on
POST /api/documents Create a new document
GET /api/documents/:id Get document details
PATCH /api/documents/:id Update document title
DELETE /api/documents/:id Delete document (owner only)
POST /api/documents/:id/share Share document with another user by email
GET /api/documents/:id/collaborators List all collaborators on a document

Comments

MethodEndpointDescription
GET /api/documents/:id/comments List all comments on a document
POST /api/documents/:id/comments Add a new comment (with optional text anchor)
PATCH /api/comments/:id Edit or resolve a comment
DELETE /api/comments/:id Delete a comment

Version History

MethodEndpointDescription
GET /api/documents/:id/versions List all saved versions of a document
POST /api/documents/:id/versions Save a version snapshot (with optional label)
POST /api/documents/:id/versions/:vid/restore Restore document to a previous version

Get Started in Minutes

Everything runs on your machine. No cloud accounts, no credit cards, no subscriptions.

Option A: Docker Compose

One command starts everything — database, servers, and frontend.

# Clone the repo and start all services
git clone <repo-url> collab-editor
cd collab-editor
make docker-up

# Open in your browser:
# Frontend:      http://localhost:3000
# Backend API:  http://localhost:4000
# Collab WS:    ws://localhost:1234
# PostgreSQL:   localhost:5432

Option B: Local Development

Run each service directly with hot-reload for development.

# Prerequisites: Node.js 20+, PostgreSQL 16+

# 1. Install dependencies
make install

# 2. Create the database
createdb collab_editor

# 3. Run database migrations
make migrate

# 4. Start all services (with hot-reload)
make dev
